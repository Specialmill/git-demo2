Git Workflows Git has a basic workflow that we will follow while we progress through a project. From the moment we first add files to the repository, and then every time we finish making changes to a section of code, we will follow this same process. Stage the changes. These changes could range from adding a new file, making a change to an existing file, or even deleting a file. Staging prepares the changes to be committed. Commit the changes to the repository. This is different from saving our files after we make our changes. When we save files after changes, we are saving the changes to the local disk (on our computer). Committing changes is saving the changes to the git repository, which is in that .git folder in our project. Commits are what allow us to see a history of each version of changes throughout the lifespan of the project.
